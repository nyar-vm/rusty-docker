# 前端模块文档

## 概述

Rusty Docker 的前端包括两个主要组件：`docker-crab-h5` 和 `docker-crab-desktop`。本文档详细介绍了前端模块的功能和实现。

## docker-crab-h5

**docker-crab-h5** 是基于 Vue.js 和 TypeScript 开发的 H5 应用，对标 Portainer，提供可视化的容器管理界面。

### 技术栈

- **框架**：Vue.js 3
- **语言**：TypeScript
- **样式**：Tailwind CSS
- **构建工具**：Vite
- **包管理**：pnpm

### 核心功能

#### 容器管理
- **容器列表**：查看所有容器的状态、名称、镜像、端口等信息
- **容器操作**：启动、停止、重启、删除容器
- **容器详情**：查看容器的详细信息，包括环境变量、网络配置、挂载卷等
- **容器日志**：实时查看容器的日志输出
- **容器终端**：通过 Web 终端进入容器执行命令

#### 镜像管理
- **镜像列表**：查看所有本地镜像的信息
- **镜像操作**：拉取、删除镜像
- **镜像详情**：查看镜像的详细信息，包括标签、大小、创建时间等
- **镜像构建**：通过 Dockerfile 构建镜像

#### 网络管理
- **网络列表**：查看所有网络的信息
- **网络操作**：创建、删除网络
- **网络详情**：查看网络的详细信息，包括子网、网关、连接的容器等

#### 存储管理
- **卷列表**：查看所有卷的信息
- **卷操作**：创建、删除卷
- **卷详情**：查看卷的详细信息，包括挂载点、大小等

#### 多容器应用编排
- **Compose 文件管理**：上传、编辑、删除 Compose 配置文件
- **服务管理**：启动、停止、重启服务
- **服务详情**：查看服务的详细信息

#### 集群管理
- **集群状态**：查看集群的整体状态
- **节点管理**：查看和管理集群节点
- **工作负载管理**：查看和管理集群中的工作负载
- **服务管理**：查看和管理集群中的服务

#### 用户界面特色
- **响应式设计**：适配不同屏幕尺寸
- **实时数据**：实时更新容器状态和日志
- **直观操作**：简洁明了的操作界面
- **深色模式**：支持深色和浅色主题
- **多语言支持**：支持中英文等多种语言

### 与 Portainer 的对比

| 功能 | docker-crab-h5 | Portainer |
|------|----------------|-----------|
| 技术栈 | Vue.js 3 + TypeScript | React |
| 部署方式 | 轻量级 H5 应用 | 容器化部署 |
| 功能完整性 | 核心容器管理功能 | 更全面的功能 |
| 性能 | 轻量快速 | 功能丰富但相对较重 |
| 自定义性 | 开源可定制 | 部分功能需付费 |
| 跨平台 | 浏览器访问，跨平台 | 容器化部署，跨平台 |

## docker-crab-desktop

**docker-crab-desktop** 是基于 Tauri 开发的桌面应用，提供更丰富的本地功能。

### 技术栈

- **框架**：Tauri
- **语言**：Rust + TypeScript
- **前端**：Vue.js
- **构建工具**：Cargo + Vite

### 核心功能

- **本地容器管理**：直接管理本地容器
- **系统集成**：与操作系统深度集成
- **托盘图标**：系统托盘图标，方便快速访问
- **桌面通知**：容器状态变更通知
- **文件系统集成**：直接访问本地文件系统

### 与 docker-crab-h5 的对比

| 功能 | docker-crab-desktop | docker-crab-h5 |
|------|---------------------|----------------|
| 部署方式 | 桌面应用 | 网页应用 |
| 系统集成 | 深度集成 | 无系统集成 |
| 性能 | 本地运行，性能更好 | 网络访问，性能依赖网络 |
| 功能范围 | 本地容器管理 | 可远程管理 |
| 跨平台 | 支持 Windows、Linux、macOS | 浏览器访问，跨平台 |

## 前端架构

### 目录结构

```
docker-crab-h5/
├── src/
│   ├── components/      # 组件
│   ├── composables/     # 组合式函数
│   ├── types/           # 类型定义
│   ├── utils/           # 工具函数
│   ├── App.vue          # 根组件
│   ├── main.ts          # 入口文件
│   └── style.css        # 全局样式
├── index.html           # HTML 入口
├── package.json         # 项目配置
├── tailwind.config.js   # Tailwind 配置
├── tsconfig.json        # TypeScript 配置
└── vite.config.js       # Vite 配置
```

### 组件设计

- **布局组件**：负责页面布局
- **功能组件**：负责具体功能实现
- **通用组件**：可复用的UI组件
- **图表组件**：数据可视化

### 状态管理

- **Pinia**：轻量级状态管理
- **LocalStorage**：本地存储
- **SessionStorage**：会话存储

### API 交互

- **Axios**：HTTP 客户端
- **WebSocket**：实时通信
- **RESTful API**：与后端服务交互

## 开发和维护

### 开发流程

1. **环境搭建**：安装 Node.js、pnpm、Vite
2. **依赖安装**：`pnpm install`
3. **开发服务器**：`pnpm dev`
4. **构建**：`pnpm build`
5. **部署**：部署到静态文件服务器

### 维护指南

- **代码风格**：遵循 Vue 代码风格指南
- **文档**：为组件和功能提供文档
- **测试**：编写单元测试和端到端测试
- **版本管理**：语义化版本控制
