# Rusty Docker 架构文档

## 1. 整体架构

Rusty Docker 采用模块化、分层的架构设计，由后端和前端两大部分组成。后端负责容器的核心功能实现，前端负责提供用户界面。

## 2. 后端架构

### 2.1 模块层次

后端架构分为以下几个层次：

1. **命令行工具层**：提供用户交互的命令行接口
2. **核心服务层**：实现容器、镜像、网络、存储等核心功能
3. **基础服务层**：提供配置管理、Docker Hub 交互、类型定义和底层运行时

### 2.2 模块详细说明

#### 2.2.1 docker-tools

**功能**：核心工具集，对标 Docker CLI、Docker Engine、Docker Compose 和 Kubernetes

**子模块**：
- **docker.rs**：Docker 命令行工具，提供与 Docker CLI 类似的命令
- **dockerd.rs**：容器运行时，类似 Docker Engine
- **docker-compose.rs**：多容器应用编排工具，类似 Docker Compose
- **kubectl.rs**：Kubernetes 命令行工具
- **kube-apiserver.rs**：Kubernetes API 服务器
- **kube-controller-manager.rs**：Kubernetes 控制器管理器
- **kube-scheduler.rs**：Kubernetes 调度器
- **kube-proxy.rs**：Kubernetes 网络代理
- **kubeadm.rs**：Kubernetes 集群管理工具

#### 2.2.2 核心服务模块

- **docker-container**：容器管理模块，负责容器的创建、启动、停止和删除
- **docker-image**：镜像管理模块，负责镜像的拉取、推送和构建
- **docker-network**：网络管理模块，负责网络的创建、配置和管理
- **docker-storage**：存储管理模块，负责卷的管理和持久化存储

#### 2.2.3 基础服务模块

- **docker-config**：配置管理模块，负责系统配置的管理
- **docker-hub**：Docker Hub 客户端，负责与 Docker Hub 的交互
- **docker-types**：类型定义模块，提供系统中使用的各种数据类型
- **rusty-docker**：核心运行时模块，提供容器运行的底层功能

### 2.3 数据流

1. **用户命令** → **docker-tools**：用户通过命令行工具输入命令
2. **docker-tools** → **核心服务模块**：命令行工具调用核心服务模块的功能
3. **核心服务模块** → **基础服务模块**：核心服务模块调用基础服务模块的功能
4. **基础服务模块** → **系统接口**：基础服务模块与操作系统交互
5. **系统接口** → **基础服务模块**：系统返回操作结果
6. **基础服务模块** → **核心服务模块**：基础服务模块返回处理结果
7. **核心服务模块** → **docker-tools**：核心服务模块返回处理结果
8. **docker-tools** → **用户**：命令行工具显示执行结果

## 3. 前端架构

### 3.1 模块层次

前端架构分为以下几个层次：

1. **UI 层**：用户界面组件
2. **业务逻辑层**：处理业务逻辑
3. **API 交互层**：与后端 API 交互

### 3.2 模块详细说明

#### 3.2.1 docker-crab-h5

**功能**：基于 Vue.js 和 TypeScript 开发的 H5 应用，对标 Portainer，提供可视化的容器管理界面

**核心组件**：
- **布局组件**：负责页面布局
- **功能组件**：负责具体功能实现
- **通用组件**：可复用的 UI 组件
- **图表组件**：数据可视化

**技术栈**：
- **框架**：Vue.js 3
- **语言**：TypeScript
- **样式**：Tailwind CSS
- **构建工具**：Vite
- **包管理**：pnpm

#### 3.2.2 docker-crab-desktop

**功能**：基于 Tauri 开发的桌面应用，提供更丰富的本地功能

**核心功能**：
- **本地容器管理**：直接管理本地容器
- **系统集成**：与操作系统深度集成
- **托盘图标**：系统托盘图标，方便快速访问
- **桌面通知**：容器状态变更通知
- **文件系统集成**：直接访问本地文件系统

**技术栈**：
- **框架**：Tauri
- **语言**：Rust + TypeScript
- **前端**：Vue.js
- **构建工具**：Cargo + Vite

### 3.3 数据流

1. **用户操作** → **UI 组件**：用户与界面交互
2. **UI 组件** → **业务逻辑**：UI 组件触发业务逻辑
3. **业务逻辑** → **API 交互**：业务逻辑调用 API
4. **API 交互** → **后端服务**：API 发送请求到后端
5. **后端服务** → **API 交互**：后端返回响应
6. **API 交互** → **业务逻辑**：API 交互处理响应
7. **业务逻辑** → **UI 组件**：业务逻辑更新 UI 状态
8. **UI 组件** → **用户**：界面显示更新结果

## 5. 核心技术实现

### 5.1 后端技术

- **Rust 语言**：提供内存安全和高性能
- **异步编程**：使用 async/await 实现异步操作
- **模块化设计**：每个功能领域独立成模块
- **错误处理**：统一的错误处理机制
- **跨平台支持**：通过条件编译实现跨平台

### 5.2 前端技术

- **Vue.js 3**：提供响应式 UI
- **TypeScript**：提供类型安全
- **Tailwind CSS**：快速构建 UI
- **Vite**：快速构建和开发
- **Pinia**：轻量级状态管理
- **Axios**：HTTP 客户端
- **WebSocket**：实时通信

## 6. 架构优势

1. **模块化**：每个功能领域独立成模块，便于维护和扩展
2. **分层设计**：清晰的分层结构，职责明确
3. **跨平台**：支持 Windows、Linux、macOS
4. **高性能**：Rust 语言提供高性能
5. **安全性**：Rust 语言的内存安全特性
6. **可扩展性**：模块化设计，易于添加新功能
7. **用户友好**：现代化的前端界面

## 7. 架构挑战

1. **跨平台兼容性**：不同操作系统的容器运行环境差异
2. **性能优化**：在保证功能的同时优化性能
3. **安全性**：确保容器的安全隔离
4. **API 兼容性**：与 Docker API 的兼容性
5. **前端性能**：确保前端界面的响应速度

## 8. 未来架构演进

1. **微服务化**：将核心服务拆分为微服务
2. **云原生集成**：与云原生技术深度集成
3. **插件系统**：支持插件扩展
4. **AI 集成**：使用 AI 优化容器管理
5. **边缘计算**：支持边缘设备的容器管理

## 9. 部署架构

### 9.1 后端部署

- **独立部署**：各个后端模块可独立部署
- **容器化部署**：可使用 Docker 容器部署
- **集群部署**：支持 Kubernetes 集群部署

### 9.2 前端部署

- **docker-crab-h5**：部署到静态文件服务器
- **docker-crab-desktop**：打包为桌面应用安装包
